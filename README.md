# Комментарии к событиям

* Основное

Система позволяет участникам событий оставлять комментарии к завершенным событиям. 
Все комментарии проходят обязательную модерацию перед публикацией.

- Модель Comment

id - уникальный идентификатор комментария
text - текст комментария (1-1000 символов)
event - событие, к которому оставлен комментарий
author - автор комментария
created - дата и время создания
status - статус комментария (PENDING, PUBLISHED, REJECTED)

- CRUD операции

Создание комментария: 
POST /users/{userId}/comments?eventId={eventId}

Удаление комментария: 
DELETE /users/{userId}/comments/{commentId}

Получение комментариев пользователя: 
GET /users/{userId}/comments?from=0&size=10

Получение опубликованных комментариев события: 
GET /comments/events/{eventId}?from=0&size=10

* Интеграция

- Public API

GET /comments/events/{eventId}

Получение опубликованных комментариев события
Пагинация: from, size
Только комментарии со статусом PUBLISHED

- Private API

POST    /users/{userId}/comments?eventId={eventId}
DELETE  /users/{userId}/comments/{commentId}
GET     /users/{userId}/comments

Создание комментариев к событиям
Удаление своих комментариев
Просмотр своих комментариев

- Admin API

GET    /admin/comments/moderation
PATCH  /admin/comments/{commentId}/moderate

Просмотр комментариев на модерации
Одобрение/отклонение комментариев

* Логика

- Создание комментария
Проверка существования пользователя и события
Проверка, что событие опубликовано
Проверка, что пользователь участвовал в событии (имел подтвержденную заявку)
Создание комментария со статусом PENDING
Сохранение в базу данных

- Модерация комментария
PATCH /admin/comments/{commentId}/moderate?approve=true
approve=true - комментарий публикуется (статус PUBLISHED)
approve=false - комментарий отклоняется (статус REJECTED)

- Удаление комментария
Только автор может удалить свой комментарий
Полное удаление из базы данных

* Дополнительно
- Модерация
Все комментарии создаются со статусом PENDING
Администраторы видят список комментариев на модерации
Администратор принимает решение: опубликовать или отклонить
Опубликованные комментарии видны всем пользователям

- Интеграция с событиями
В EventFullDto добавлено поле commentCount
Автоматический подсчет опубликованных комментариев
Не требуется хранение счетчика в таблице событий

- Ограничения
Максимальная длина комментария: 1000 символов
Минимальная длина: 1 символ
Комментарии только к завершенным событиям
Комментарии только от участников событий
Все комментарии проходят модерацию
